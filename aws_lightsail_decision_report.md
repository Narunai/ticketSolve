## 📌 1. บทวิเคราะห์ผลกระทบเรื่องขนาดไฟล์แนบ (Attachment Storage Analysis)

การเพิ่มสิทธิ์ให้แนบไฟล์ในแต่ละตั๋ว (เช่น รูปภาพสกรีนช็อตข้อผิดพลาด, ไฟล์เอกสาร PDF, หรือไฟล์ log) โดยจำกัดขนาดสูงสุดไว้ที่ **10 MB ต่อตั๋ว** ส่งผลให้เราต้องคำนวณพื้นที่จัดเก็บข้อมูล (SSD Storage) บนเซิร์ฟเวอร์เพื่อให้รองรับการเก็บข้อมูลในระยะยาวได้อย่างเสถียรและประหยัดงบประมาณที่สุด

### สมมติฐานในการคำนวณความจุ (Data Model Assumptions)
1. **ระบบปฏิบัติการและซอฟต์แวร์พื้นฐาน (OS Overhead)**: กินพื้นที่ประมาณ **7 GB** (ระบบปฏิบัติการ Ubuntu + ไลบรารี Python/Django + ฐานข้อมูลหลัก SQLite + ไฟล์ล็อกระบบ)
2. **ขนาดไฟล์แนบเฉลี่ย (Average File Size)**: แม้จะจำกัดที่ 10 MB แต่ตามสถิติจริง ไฟล์แนบเฉลี่ยส่วนใหญ่จะอยู่ที่ประมาณ **4 MB** (ภาพสกรีนช็อตบีบอัด หรือไฟล์เอกสารทั่วไป)
3. **ปริมาณตั๋วที่มีไฟล์แนบ**: ประเมินว่าจะมีตั๋วที่มีการอัปโหลดไฟล์แนบประมาณ **50%** ของตั๋วทั้งหมดในระบบ

## 💡 3. เหตุผลในการแนะนำแพ็คเกจ $12 USD / เดือน (Best Value Pick)

1. **ประสิทธิภาพความเร็ว (Compute Performance)**:
   การมี **2 vCPUs และ 2 GB Memory** ช่วยป้องกันอาการหน่วยความจำเต็ม (Out of Memory) ได้เป็นอย่างดี โดยเฉพาะในจังหวะที่มีการอัปโหลดไฟล์พร้อมกัน หรือช่วงสัปดาห์สิ้นเดือนที่มีการรันรายงานสรุปตั๋วแบบ PDF ยิงส่งเมลออกไปหาพนักงานจำนวนมาก
2. **ความคุ้มค่าด้านพื้นที่ (Storage Lifespan)**:
   เมื่อเหลือพื้นที่ใช้งานจริง **53 GB** และคำนวณที่อัตราตั๋วใหม่ 300 ใบต่อเดือน (แนบไฟล์ 50% เฉลี่ยใบละ 4 MB = 600 MB ต่อเดือน) ระบบจะสามารถเปิดรันงานได้อย่างราบรื่นยาวนานกว่า **7 ปี** โดยไม่ต้องอัปเกรดแพ็คเกจหรือพื้นที่เพิ่ม
3. **Bandwidth โอนถ่ายข้อมูลเหลือเฟือ**:
   โควตา **3 TB Data Transfer** ขาออก ช่วยการันตีว่ายอดบิลสิ้นเดือนจะไม่มีค่าใช้จ่ายแปลกปลอม ถึงแม้ผู้ใช้จะดาวน์โหลดและเปิดดูไฟล์รูปภาพขนาดใหญ่บ่อยครั้งก็ตาม

## ⚙️ 4. คำแนะนำในการบริหารจัดการพื้นที่ในอนาคต (Storage Maintenance Tips)

* **ระบบบีบอัดภาพ (Image Compression)**: หากระบบจัดเก็บไฟล์เริ่มมีขนาดใหญ่ แนะนำให้เขียนมิดเดิลแวร์บีบอัดไฟล์ภาพอัตโนมัติก่อนบันทึกลงดิสก์
* **ย้ายไปเก็บที่ AWS S3 (Scalable Strategy)**: หากแอปพลิเคชันขยายตัวจนพื้นที่ 60 GB เริ่มไม่พอในอนาคต เราสามารถย้ายระบบจัดเก็บไฟล์แนบจากในเครื่อง Lightsail Instance ไปฝากไว้ที่ **Amazon S3** ได้โดยจ่ายเฉพาะค่าพื้นที่ฝากส่ง ซึ่งประหยัดและรองรับขนาดได้ไม่จำกัด
