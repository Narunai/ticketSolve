# 👥 คู่มือสรุปบทบาทและสิทธิ์ผู้ใช้งานในระบบ TicketSolve (User Roles Guide)

เอกสารนี้ระบุรายละเอียด ขอบเขตข้อมูลที่เข้าถึงได้ และสิทธิ์การใช้งาน (Role-Based Access Control - RBAC) ของบทบาทสมาชิกทั้งหมดในระบบ TicketSolve เพื่อเป็นคู่มืออ้างอิงความปลอดภัย

---

## 📊 ตารางเปรียบเทียบสิทธิ์การใช้งาน (Role Comparison Table)

| บทบาทผู้ใช้ (Role) | ขอบเขตข้อมูลที่มองเห็น (Scope) | สิ่งที่ทำได้ในระบบ (Capabilities) | ข้อจำกัดสิทธิ์ (Restrictions) |
| :--- | :--- | :--- | :--- |
| **1. System Admin**<br>*(แอดมินสูงสุดส่วนกลาง)* | **ทั้งหมดในระบบ**<br>(ทุกบริษัท) | • สร้าง/แก้ไข/ลบ ทุกบริษัทในระบบ<br>• จัดการผู้ใช้ทุกคนในระบบ (ทุกบทบาท)<br>• ดู/จัดการตั๋วทุกใบของทุกบริษัท<br>• จัดการเซิร์ฟเวอร์ SMTP (เพิ่ม/ลบ/แก้ไข/เปิดใช้งาน)<br>• ดูพรีวิว PDF และกดส่งรายงานหาใครก็ได้ในระบบ<br>• ดู Log การเข้าถึงและประวัติการส่งเมลทั้งหมด | **ไม่มี** (มีสิทธิ์สูงสุดเสมือน Superuser ในระบบ) |
| **2. System Sub-Admin**<br>*(แอดมินย่อยส่วนกลาง)* | **ทั้งหมดในระบบ**<br>(ทุกบริษัท) | • ดูและจัดการแก้ตั๋วของทุกบริษัทได้ทั้งหมด<br>• ดูรายชื่อบริษัทและรายชื่อสมาชิกเพื่อช่วยเหลือผู้ใช้<br>• สร้าง/แก้ไขสมาชิกได้เฉพาะสิทธิ์ระดับลูกค้า<br>• ดูพรีวิว PDF และกดส่งรายงานหาใครก็ได้ในระบบ<br>• ดู Log การเข้าถึงและประวัติการส่งเมลทั้งหมด | • **ไม่สามารถ** เข้าถึงหน้าตั้งค่าระบบ SMTP ได้<br>• **ไม่สามารถ** สร้าง/แก้ไขสิทธิ์สมาชิกที่เป็น System Admin หรือ Sub-Admin คนอื่นๆ ได้ |
| **3. Client Admin**<br>*(แอดมินของบริษัทลูกค้า)* | **เฉพาะในบริษัทตนเอง** | • จัดการเพิ่ม/แก้ไขสมาชิกพนักงานในบริษัทตนเองได้<br>• ดู/ติดตาม/อัปเดตสถานะตั๋วทุกใบในบริษัทตนเอง<br>• ดูพรีวิวและดาวน์โหลดรายงาน PDF ของบริษัทตนเองได้<br>• ดู Log การเข้าถึงและประวัติเมลเฉพาะในบริษัทตนเอง | • **ไม่สามารถ** มองเห็นตั๋ว สมาชิก หรือข้อมูลของบริษัทอื่นได้เลย<br>• **ไม่สามารถ** จัดการเซิร์ฟเวอร์ SMTP ได้ |
| **4. Client User**<br>*(พนักงานทั่วไปของลูกค้า)* | **เฉพาะตั๋วของตัวเอง** | • เขียนส่งใบตั๋วแจ้งปัญหาใหม่เข้าระบบ<br>• ติดตามและตอบแชทโต้ตอบ (Comments) ในตั๋วของตัวเอง<br>• กดปิดตั๋วของตัวเองเมื่อแก้ไขปัญหาเสร็จสิ้นแล้ว | • **ไม่สามารถ** มองเห็นตั๋วของเพื่อนร่วมงานคนอื่นได้ (ยกเว้นเป็นแอดมินบริษัท)<br>• **ไม่สามารถ** เข้าถึงรายงาน PDF หรือการจัดการผู้ใช้ใดๆ ได้ |

## 💡 ตัวอย่างสถานการณ์จำลองสิทธิ์การเข้าใช้งาน (Use-Case Scenarios)

### สถานการณ์ที่ 1: การแจ้งปัญหาของพนักงานต่างบริษัท
* **พนักงาน A (Client User)** สังกัด **Company A** สร้างใบตั๋วแจ้งเรื่อง "Internet ใช้งานไม่ได้"
* **พนักงาน B (Client User)** สังกัด **Company B** ล็อกอินเข้าระบบ ➔ หน้าแดชบอร์ดและหน้ารายการตั๋วของพนักงาน B จะไม่ขึ้นตั๋วนี้ และไม่สามารถคลิกเปิดดูได้เนื่องจากระบบคัดกรอง Query กั้นข้อมูลไว้ในระดับฐานข้อมูล

### สถานการณ์ที่ 2: การดูแลของแอดมินบริษัท (Client Admin)
* **แอดมินบริษัท A (Client Admin)** เข้าหน้าจอ **รายงานประจำเดือน** ➔ จะสามารถดูรายการพรีวิว PDF รายงานประจำเดือนของบริษัท A และกดส่งอีเมลหาพนักงานภายในบริษัท A ได้ทุกคน แต่จะมองไม่เห็นฟังก์ชันการสลับไปพรีวิวรายงานของบริษัท B เลย

### สถานการณ์ที่ 3: ข้อจำกัดความปลอดภัยของ System Sub-Admin
* **แอดมินส่วนกลางย่อย (System Sub-Admin)** เข้าหน้าจอควบคุม ➔ สามารถมองเห็นและเข้าไปช่วยเหลือตอบตั๋วของทั้งบริษัท A และบริษัท B ได้ตามปกติเพื่อทำหน้าที่สนับสนุนทางเทคนิค
* แต่หากพยายามพิมพ์ URL เพื่อขอเข้าถึง `/settings/` (หน้าตั้งค่า SMTP) หรือพยายามกดสร้าง/แก้ไขสิทธิ์แอดมินระดับสูง ➔ ระบบจะตอบกลับด้วยหน้าข้อผิดพลาด **403 Forbidden** ทันที เพื่อป้องกันข้อมูลการส่งเมลหลักรั่วไหล
