## 🎯 1. ภาพรวมของระบบ (Project Overview)
**TicketSolve** คือระบบจัดการแจ้งปัญหาการทำงานทางเทคนิคแบบแยกข้อมูลตามรายบริษัท (**Multi-tenant Helpdesk Ticket System**) พัฒนาขึ้นโดยใช้เฟรมเวิร์ก Django เพื่อให้ผู้ให้บริการส่วนกลาง (เช่น แผนกไอทีกลาง หรือบริษัท One IT) สามารถดูแลและสนับสนุนการแจ้งปัญหาของหลายๆ บริษัทลูกค้าได้พร้อมกันอย่างเป็นระเบียบ ปลอดภัย และมีการปิดกั้นข้อมูล (Data Isolation) ระหว่างองค์กรอย่างสมบูรณ์แบบ

## 🛠️ 2. รายงานฟังก์ชันและคุณสมบัติทั้งหมดในระบบ (Complete Feature Catalog)

### 1. ระบบแยกข้อมูลองค์กรและพนักงาน (Multi-tenant & Data Isolation)
* **การแยกข้อมูลตั๋ว (Ticket Isolation)**: พนักงานและผู้ดูแลระบบของบริษัทหนึ่งๆ จะไม่สามารถมองเห็น ดึงข้อมูล หรือแก้ไขตั๋วแจ้งปัญหาของบริษัทอื่นได้
* **การคุมขอบเขตสิทธิ์ (Scope Lockdown)**: บังคับใช้เงื่อนไขตรวจสอบสิทธิ์ความปลอดภัยในระดับ View เพื่อให้สิทธิ์ดึงข้อมูลทำงานเฉพาะองค์กรของตนเองเท่านั้น แม้จะมีพยายามยิง URL ข้ามบริษัทระบบจะปฏิเสธการเข้าถึงทันที (403 Forbidden)

### 2. ระบบจัดการระดับสิทธิ์ผู้ใช้งาน (Role-Based Access Control - RBAC)
ระบบจำแนกผู้ใช้แบ่งสิทธิ์ออกเป็น 4 ระดับหลัก เพื่อรองรับการทำงานในระดับองค์กรและส่วนกลาง:
* **System Administrator (ส่วนกลาง)**: มีสิทธิ์สูงสุดในการดูแลระบบทั้งหมด จัดการโดเมน ตั๋ว บริษัท และปรับแต่งเซิร์ฟเวอร์ SMTP
* **System Sub-Administrator (ส่วนกลางระดับย่อย)**: มีสิทธิ์ระดับสูงช่วย System Admin จัดการตั๋วและผู้ใช้ของทุกบริษัทเพื่อซัพพอร์ตงานทั่วไป แต่**ไม่มีสิทธิ์** เข้าถึงหน้าตั้งค่าระบบ SMTP และไม่สามารถแก้ไขสิทธิ์ของ System Admin หรือผู้ดูแลคนอื่นได้
* **Client Administrator (แอดมินบริษัทลูกค้า)**: มีสิทธิ์ดูแลตั๋ว ดูรายงาน PDF และจัดการพนักงานเฉพาะภายในบริษัทสังกัดตนเองเท่านั้น
* **Client User (พนักงานบริษัทลูกค้า)**: มีสิทธิ์แจ้งตั๋วปัญหาใหม่ ติดตาม และปิดตั๋วของตนเองเท่านั้น

### 3. ระบบวงจรชีวิตของตั๋วแจ้งปัญหา (Ticket Lifecycle Management)
* **การสร้างตั๋ว (Ticket Creation)**: ระบุหัวข้อ ปัญหา รายละเอียด ระดับความสำคัญ และประเภทหมวดหมู่ปัญหา
* **หมวดหมู่ปัญหา (Ticket Categories)**: รองรับการจัดหมวดหมู่ตั๋วออกเป็น 5 หมวดหมู่หลัก ได้แก่
  1. **Hardware** (อุปกรณ์คอมพิวเตอร์ / เครื่องมือ)
  2. **Software** (โปรแกรม / ซอฟต์แวร์ระบบ)
  3. **Network / Internet** (ระบบเครือข่าย / อินเทอร์เน็ต)
  4. **Account / Login** (บัญชีผู้ใช้งาน / การเข้าสู่ระบบ)
  5. **Other** (อื่นๆ)
* **การอัปเดตและติดตาม**: ผู้ดูแลระบบสามารถเปลี่ยนระดับความสำคัญ (Priority: Low, Medium, High, Urgent) และสถานะการซ่อมบำรุง (Status: New, In Progress, Resolved, Closed)
* **การตอบกลับบันทึกข้อความ (Comments/Feedback)**: พนักงานและช่างไอทีสามารถพิมพ์บันทึกข้อความโต้ตอบกันภายในตั๋วแต่ละใบเพื่อติดตามความคืบหน้างาน

### 4. ระบบออกรายงานประจำเดือนในรูปแบบ PDF (Monthly PDF Report System)
* **HTML-to-PDF Engine**: พัฒนาระบบรวมรวมข้อมูลสถิติตั๋วประจำเดือนแปลงเป็นไฟล์ PDF สวยงาม ผ่านไลบรารี `xhtml2pdf`
* **ป้องกันฟอนต์สี่เหลี่ยม (Font Encoding Fixed)**: ติดตั้งฟอนต์ **Tahoma** บนเซิร์ฟเวอร์เพื่อให้แสดงผลข้อความภาษาไทยและภาษาอังกฤษได้ถูกต้องคมชัด ไม่เป็นกล่องสี่เหลี่ยม
* **ดูตัวอย่างรายงาน (PDF Preview)**: ผู้ดูแลระบบสามารถเปิดดูตัวอย่างไฟล์ PDF ล่าสุดบนเบราว์เซอร์ได้ทันทีก่อนส่งจริง
* **ส่งรายงานด่วนทันที (On-Demand Dispatcher)**: สั่งส่งรายงานให้ทุกคนในบริษัท หรือเลือกส่งเฉพาะบุคคลรายรายชื่อที่ต้องการได้ทันที
* **ระบบเลือกอีเมลผู้ส่ง (Send From Account)**: สามารถเลือกได้บนหน้า Dashboard รายงานว่าจะใช้บัญชีอีเมล SMTP คอนฟิกตัวใดในการยิงส่งไฟล์ PDF ออกไป
* **ประวัติการอ่านและการจัดส่ง (Audit Logs)**:
  * **Report View Logs**: เก็บบันทึกประวัติว่าใครเปิดอ่านไฟล์ PDF รายงานของบริษัทใด เมื่อไหร่
  * **Email Notification Logs**: บันทึกรายงานการจัดส่งอีเมลและสถานะความสำเร็จอย่างละเอียด

### 5. ระบบจัดการเซิร์ฟเวอร์ SMTP บนหน้าเว็บ (Dynamic SMTP Configurations)
* **การตั้งค่าผ่านเว็บ (Settings UI)**: แอดมินหลักสามารถสร้าง ลบ สลับเปิด/ปิด หรือแก้ไข คอนฟิกเมล SMTP ได้ผ่านเว็บแอปพลิเคชันโดยตรงไม่ต้องเปิดไฟล์โค้ด
* **Presets บัญชีผู้ให้บริการด่วน**: มีระบบสุ่มกรอกค่าโฮสต์และพอร์ตให้อัตโนมัติสำหรับ Google Gmail, Microsoft Outlook, และ Simulation
* **การันตี Active เซิร์ฟเวอร์เดี่ยว**: ระบบบังคับให้ใช้งานเชื่อมต่อ SMTP บัญชีส่งออกจริงได้เพียงทีละ 1 เซิร์ฟเวอร์เท่านั้น เพื่อความเสถียร
* **คู่มือการใช้งาน (Integrated Guide)**: แสดงคู่มือวิธีการขอรับ App Password 16 หลักจาก Google และ Microsoft แบบแยกแท็บแสดงผลไว้ข้างฟอร์มสำหรับช่วยแอดมินตั้งค่า

### 6. ความปลอดภัยและการปรับปรุงการใช้งาน (UX/UI & Security Hardening)
* **ระบบสลับเปลี่ยนภาษา (Bilingual TH/EN Switcher)**: หน้าจอส่วนติดต่อผู้ใช้งานหลักทั้งหมด (Dashboard, ฟอร์มเปิดตั๋ว, รายละเอียดตั๋ว, สไลด์บาร์, แผงควบคุม) รองรับการแสดงผล **ภาษาไทย และ ภาษาอังกฤษ (TH/EN)** สลับเปลี่ยนได้ทันทีผ่านสวิตช์บนแถบ Header 
* **ล็อกอินยืดหยุ่น (Dual-Field Login)**: ปรับปรุงหน้าเข้าใช้งานให้รองรับการป้อนข้อมูลได้ทั้ง **ชื่อผู้ใช้งาน (Username)** หรือ **อีเมล (Email)** คู่กับรหัสผ่าน
* **ปุ่มแสดงรหัสผ่าน (Password Visibility Toggle)**: เพิ่มปุ่มตา (Eye Icon) ช่วยซ่อน/แสดงอักขระรหัสผ่านในทุกกล่องป้อนข้อมูล
* **ปรับปรุง Layout บนมือถือ**: ตั้งค่า Z-index ของแถบส่วนหัวเว็บป้องกันเมนูเลือกโหมด/ธีมสีไม่ให้การ์ดในเนื้อหาแสดงทับบนสมาร์ทโฟน
